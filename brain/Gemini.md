# Gemini.md (Hermes運用メモ)

## Purpose

Gemini系モデルをHermesで使う際の、実装・運用上の注意点をまとめる。

## Model Usage

- 役割分担: 高速探索は軽量モデル、最終判断は高品質モデルを使い分ける。
- 出力方針: 検証可能な根拠を優先し、断定は確認できる範囲に限定する。
- 安全方針: 危険行為・違法行為・秘密漏えいにつながる要求は拒否または安全代替を提示する。

## Tool Schema Notes

- スキーマは単純で明示的に保つ（トップレベル object + properties）。
- nullableより optional を優先し、実装側で入力検証を行う。
- 互換性を崩す拡張は段階的に行い、既存ツール呼び出しを壊さない。

## Operational Notes

- 重要変更は `_docs/` に記録する。
- マルチエージェント環境では他者変更を破壊せず、担当範囲に集中する。
