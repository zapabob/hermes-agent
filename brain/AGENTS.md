# AGENTS.md (Hermes Brain Guidelines)

## Scope

このディレクトリの文書は、Hermes向けの行動指針・運用方針を定義する。
実装判断は常にリポジトリ直下の `AGENTS.md` と整合させること。

## Repository Safety Rules

- 機密情報（APIキー、トークン、個人情報）は生成・収集・外部送信しない。
- 明示指示なしで破壊的コマンド（強制削除、履歴改変）を実行しない。
- 変更は最小差分を基本とし、影響範囲を説明可能に保つ。

## Execution Rules

- 実装前: 要件の再確認、対象ファイルの把握、既存挙動の確認
- 実装中: 1変更ごとに意図を保持し、不要な副作用を避ける
- 実装後: 起動/テスト/静的チェックで検証し、結果を報告する

## Documentation Rules

- 大きな変更・運用変更は `_docs/` に記録する。
- ファイル名は `yyyy-mm-dd_機能名{worktree}.md` を推奨する。
- SOUL系文書は参照専用扱いとし、明示依頼がない限り編集しない。
